# tiles-performance-comparison
## 東京圏における建築物3DTilesモデルの最適化前後比較

---

## 概要
本プロジェクトでは、プラトーからダウンロードした建築物の3D Tilesモデルを対象に、表示パフォーマンスの改善を行いました。  
最大50MBの大規模モデルを効率的に読み込めるよう、タイル分割・LOD・テクスチャ圧縮などの最適化手法を適用しています。

---

## 背景
- プラトーの3D Tilesを読み込む際、ブラウズがスムーズでなく、カクつきが発生  
- 最大50.6MBのタイルモデルにより、読み込み時にスレッドが一時停止する現象が確認された

---

## 技術的アプローチ
１. **プラトーの3D Tiles直接最適化**  
　- 最大タイルはルートノードであり、分割不可 → 断念  

２. **CityGMLをGLTFに変換して3D Tiles再生成**  
　- 公式ツールでは全メッシュ統合 → 建物単位で分割不可 → 断念  

３. **FBX / OBJ → GLTF → タイル分割による3D Tiles生成**  
　- 実験で有効性を確認  
　- この方法を採用

---

## 最適化ポイント
- **タイルサイズの制御**  
　各タイルのモデル面数を5,000以下に制限  

- **KTX2テクスチャ圧縮**  
　VRAM使用量を約80%削減、メモリ使用量を約30%削減  

- **LOD（Level of Detail）の導入**  
　粗モデルと精密モデルを活用  
　遠距離：LOD1のみ  
　近距離：LOD2を読み込む

---

## 今後の改善予定
- **LODの違和感改善**  
　LOD1とLOD2の差が大きいため、中間LODを生成して自然な遷移を実現予定  

- **LODとタイルの整合性**  
　LOD1のみのエリアとLOD1+LOD2混在エリアを分離してタイル分割予定  

- **過小タイルの統合**  
　小さいタイルを隣接タイルと統合し、リクエスト数を削減・読み込み速度を改善

---

## 成果
- タイルサイズ制御＋LOD＋テクスチャ圧縮により、ブラウズのカクつきが大幅に軽減  
- 遠距離から近距離までスムーズに表示可能  
- VRAM・メモリ使用量の大幅削減
